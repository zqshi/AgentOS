## **「智核」平台 - "智能体市场与应用"模块详细产品需求文档**

**文档编号**:ZH-PRD-M5
**模块名称**:智能体市场与应用
**产品定位**:企业智能体的发现、订阅、配置与管理中心,让每位员工拥有个性化的AI助手团队。
**核心目标**:将「灵枢」智能体工厂培育的各类智能体,以用户友好的方式呈现和交付给员工,让智能体从"技术概念"变为"随手可用的数字同事",并提供全生命周期的订阅、配置、监控与反馈机制。

### **一、模块全景与核心价值**

#### **1.1 要解决的根本问题**
企业AI应用面临三大挑战:
1. **发现困难**:智能体散落在各处,员工不知道有哪些AI助手可用,更不知道哪个适合自己的需求。
2. **使用门槛高**:技术人员开发的智能体往往缺乏友好界面,业务人员难以配置和使用。
3. **缺乏信任与反馈**:员工不知道智能体的能力边界和可靠性,使用后无法有效反馈,导致智能体质量无法迭代提升。

智能体市场模块通过**统一商店**、**低门槛订阅与配置**、**透明化评价与监控**,让智能体真正服务于每一位员工。

#### **1.2 核心设计理念**
1. **App Store范式 (Marketplace Paradigm)**:借鉴消费级应用商店的成功经验,让智能体的发现和订阅如同下载App一样简单。
2. **信任优先 (Trust-First)**:通过详细的能力说明、使用案例、用户评价、官方认证,建立员工对智能体的信任。
3. **个性化赋能 (Personalized Empowerment)**:每个员工根据角色和需求,组建自己的专属AI助手团队。
4. **持续进化 (Continuous Evolution)**:智能体不是一次性交付,而是通过用户反馈和数据驱动持续优化升级。

#### **1.3 核心用户角色**
| 角色 | 核心使用场景 | 关键价值获得 |
| :--- | :--- | :--- |
| **普通员工(智能体消费者)** | 浏览和发现可用智能体;订阅适合自己的AI助手;配置和使用智能体;反馈使用体验。 | 快速找到能解决自己工作痛点的AI助手;零开发成本使用强大AI能力;通过反馈推动工具改进。 |
| **业务专家/领域Leader** | 发现适合团队的专业智能体;为团队统一订阅和配置;监控团队使用情况。 | 为团队配备专业AI工具,提升整体效率;通过数据了解团队工作模式。 |
| **智能体开发者(内部)** | 发布自己开发的智能体;查看使用数据和用户反馈;迭代优化智能体。 | 让自己的作品被更多人使用;获得明确的改进方向;建立个人影响力。 |
| **AI产品经理/运营** | 策划和运营智能体商店;分析智能体使用数据;推荐优质智能体;下架问题智能体。 | 优化智能体生态;发现用户需求缺口;驱动智能体质量提升。 |
| **管理者** | 了解组织智能体使用情况;评估AI投资回报;识别高价值智能体和应用场景。 | 获得AI赋能的量化数据;优化资源投入;推广最佳实践。 |

### **二、详细功能需求**

#### **M5.1 智能体商店(发现与订阅中心)**

**目标**:提供直观、丰富、可信赖的智能体发现和订阅平台,让员工轻松找到并获取所需AI能力。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 商店首页与导航** | • **精选推荐区**:首页顶部展示精选智能体:<br>  - **官方推荐**:平台运营方推荐的高质量、高适用性智能体。<br>  - **新上架**:最近发布的新智能体(带"新"标识)。<br>  - **热门榜单**:按订阅量、使用量、好评率排序的热门智能体。<br>  - **为你推荐**:基于用户角色、部门、历史使用记录的个性化推荐。<br>• **分类导航**:按业务场景和功能类型分类:<br>  - **按场景分类**:销售助手、研发工具、财务分析、人力资源、客户服务、项目管理等。<br>  - **按能力分类**:文本生成、数据分析、流程自动化、知识问答、审核辅助等。<br>  - **按部门分类**:研发、销售、市场、财务、行政等。<br>• **搜索与筛选**:<br>  - **智能搜索框**:支持按名称、功能描述、场景搜索,AI理解用户需求推荐相关智能体。<br>  - **高级筛选器**:按认证类型(官方/第三方)、评分、使用人数、更新时间、是否免费筛选。 |
| **2. 智能体详情页** | • **核心信息区**:<br>  - **名称与图标**:智能体名称、精美图标(建立视觉识别)。<br>  - **一句话介绍**:简洁描述核心功能(如"自动审查合同风险条款的法务助手")。<br>  - **认证标识**:官方认证、部门推荐、测试版等标识。<br>  - **关键指标**:订阅数、使用次数、平均评分(星级)、成功率。<br>• **详细描述**:<br>  - **功能说明**:详细描述智能体的能力边界、适用场景、典型任务。<br>  - **使用案例**:展示3-5个真实使用案例(如"销售团队使用后,客户跟进效率提升40%")。<br>  - **能力演示**:提供视频或交互式演示,让用户直观了解智能体如何工作。<br>• **技术信息(可折叠)**:<br>  - 底层模型、依赖的「灵枢」能力(如知识图谱、规则引擎)、数据来源、隐私说明。<br>• **用户评价区**:<br>  - 展示用户评分和评论(支持排序:最新/最高分/最有用)。<br>  - 每条评价显示评分(星级)、评论内容、评论者角色、时间。<br>  - 支持点赞和回复评论。<br>• **版本历史与更新日志**:展示智能体的版本迭代历史和每个版本的更新内容。<br>• **订阅按钮**:<br>  - 明显的"立即订阅"按钮(如已订阅则显示"已订阅"和"打开使用")。<br>  - 显示订阅后会获得的权益和可能需要的配置。 |
| **3. 订阅与权限管理** | • **一键订阅**:点击订阅按钮后:<br>  - 免费智能体直接完成订阅,自动添加到"我的智能体"列表。<br>  - 付费或需审批的智能体,引导进入申请流程。<br>• **订阅类型**:<br>  - **个人订阅**:仅当前用户可用。<br>  - **团队订阅**:Team Leader可为整个团队统一订阅,团队成员自动获得。<br>• **权限说明**:订阅时明确告知智能体需要的权限(如"需要访问您的客户数据"、"需要发送邮件权限"),用户需确认授权。<br>• **试用机制**:部分智能体提供"7天免费试用"或"10次免费调用",让用户先体验再决定。 |
| **4. 智能体生态运营** | • **专题推荐**:运营方定期策划专题(如"销售效能月"、"研发提效工具集"),聚合相关智能体推广。<br>• **新手指引**:新用户首次进入商店,提供引导教程:"如何找到适合你的智能体" → "如何订阅和配置" → "如何反馈问题"。<br>• **智能体排行榜**:<br>  - **热度榜**:最近一周使用量最高的智能体。<br>  - **好评榜**:评分最高且评价数>50的智能体。<br>  - **新秀榜**:最近上架且表现优异的智能体。<br>• **缺失需求收集**:商店底部提供"找不到你需要的?提交需求"入口,收集用户需求,引导智能体开发。 |

#### **M5.2 我的智能体工作台(个人AI助手团队管理中心)**

**目标**:为每位员工提供统一界面,管理和使用其订阅的所有智能体,让多个AI助手协同工作。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 我的智能体列表** | • **列表展示**:以卡片或列表形式展示用户已订阅的所有智能体:<br>  - **智能体信息**:图标、名称、简介、订阅时间。<br>  - **使用统计**:我的使用次数、最近使用时间、我的评分。<br>  - **快捷操作**:打开使用、配置、取消订阅、查看详情。<br>• **分组管理**:<br>  - 支持将智能体分组(如"日常工具"、"项目专用"、"学习助手"),自定义分组名称。<br>  - 支持收藏(星标)常用智能体,置顶显示。<br>• **状态标识**:<br>  - 显示智能体状态(运行正常/维护中/已过期/有新版本)。<br>  - 有新版本时,提示"点击更新"。 |
| **2. 智能体配置与个性化** | • **配置入口**:每个智能体卡片提供"配置"按钮,进入配置界面。<br>• **配置选项**(根据智能体类型不同):<br>  - **工作范围设置**:如"合同审查助手"可设置"只审查金额>100万的合同"。<br>  - **数据源选择**:选择智能体可访问的知识库范围、团队空间、项目。<br>  - **通知偏好**:设置智能体完成任务后的通知方式(即时通知/每日汇总/不通知)。<br>  - **个性化参数**:如"内容生成助手"可设置输出风格(正式/轻松)、长度偏好(简洁/详细)。<br>• **权限管理**:查看和调整智能体的访问权限(如临时关闭某项权限)。<br>• **配置模板**:对于复杂配置,提供预设模板(如"销售场景"、"技术场景"),一键应用。 |
| **3. 智能体协同使用** | • **任务编排**:高级用户可创建"智能体工作流":<br>  - 定义任务流程(如"数据提取助手 → 分析师助手 → 报告生成助手"),智能体按顺序协同工作。<br>  - 通过可视化拖拽界面配置流程,无需编程。<br>• **一对多调用**:在对话中,可同时@多个智能体协作完成复杂任务(如"@数据分析师 @文案助手,帮我生成Q4业绩报告")。<br>• **智能体推荐**:当用户使用某个智能体时,系统推荐"常与此智能体搭配使用的其他智能体"。 |
| **4. 使用历史与记录** | • **调用历史**:展示每个智能体的使用历史:<br>  - **任务记录**:每次调用的任务描述、输入、输出、时间、耗时、状态(成功/失败)。<br>  - **思维链查看**:点击某次任务,可查看智能体的完整推理过程(透明化黑盒)。<br>• **收藏与复用**:<br>  - 对满意的输出结果,可点击"收藏",保存到个人知识库。<br>  - 可基于历史任务一键"再次执行"或"修改参数后执行"。<br>• **数据导出**:支持导出使用历史数据(CSV/Excel),用于个人分析或汇报。 |

#### **M5.3 智能体监控与反馈**

**目标**:让智能体的运行状态透明可见,收集用户反馈,驱动智能体持续优化。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 实时运行状态监控** | • **执行中任务**:在"我的智能体"页面,实时显示正在执行的任务:<br>  - 任务名称、进度条、预计剩余时间、当前步骤说明。<br>  - 支持点击查看详细执行日志(调用了哪些接口、查询了哪些数据)。<br>• **异常告警**:<br>  - 智能体执行失败时,立即通知用户,并提供失败原因(如"数据源访问超时"、"权限不足")。<br>  - 提供"重试"或"联系支持"选项。<br>• **性能指标**:展示智能体的性能统计:<br>  - 平均执行时长、成功率、最近7天趋势图。<br>  - 与其他用户的平均水平对比(如"你的成功率95%,高于平均水平10%")。 |
| **2. 用户反馈系统** | • **任务级反馈**:每次智能体完成任务后,提示用户评价:<br>  - **快捷评价**:👍/👎 + "有用"/"不准确"/"太慢"等快捷标签。<br>  - **详细反馈**:可选填文本框,描述问题或建议。<br>• **智能体整体评价**:<br>  - 在智能体详情页,用户可提交整体评分(1-5星)和评论。<br>  - 鼓励用户分享使用心得和改进建议。<br>• **问题报告**:提供"报告问题"入口,用户可提交:<br>  - 问题类型(功能错误/性能问题/权限问题/改进建议)。<br>  - 问题描述、截图、相关任务ID。<br>  - 系统自动关联该智能体的开发团队,创建工单。<br>• **反馈可见性**:<br>  - 用户可查看自己提交的反馈处理状态(已提交/处理中/已解决/已忽略)。<br>  - 智能体更新时,若某个功能改进来自用户反馈,在更新日志中@感谢贡献者。 |
| **3. 智能体健康度报告(面向开发者)** | • **数据看板**:智能体开发者可查看专属数据看板:<br>  - **使用数据**:订阅数、活跃用户数、调用次数、使用趋势。<br>  - **质量数据**:成功率、平均响应时间、用户满意度(好评率)。<br>  - **反馈汇总**:用户反馈分类统计、高频问题、改进建议词云。<br>• **对比分析**:与同类智能体的关键指标对比,识别差距。<br>• **优化建议**:系统基于数据分析,给出具体优化建议(如"响应时间比同类慢30%,建议优化数据查询逻辑")。 |

#### **M5.4 智能体生命周期管理(面向运营和管理者)**

**目标**:确保智能体生态健康,持续提供高质量服务。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 智能体发布与审核** | • **发布流程**:智能体开发者(或「灵枢」后台)提交智能体发布申请:<br>  - **基础信息**:名称、图标、分类、描述、使用说明、案例。<br>  - **技术信息**:依赖资源、权限需求、性能指标承诺。<br>  - **测试报告**:提供测试数据(功能覆盖、性能测试、安全测试)。<br>• **审核机制**:<br>  - **自动审核**:系统自动检查基础合规性(如描述完整性、权限合理性、安全漏洞扫描)。<br>  - **人工审核**:AI产品经理或领域专家进行功能验证、质量评估、适用性审核。<br>  - **审核标准**:明确的审核清单(功能正确性、描述准确性、用户友好性、安全性)。<br>• **上架与分发**:审核通过后,智能体正式上架商店,并根据策略推送给目标用户。 |
| **2. 智能体版本管理** | • **版本发布**:智能体更新时,开发者提交新版本:<br>  - 填写版本号、更新内容、兼容性说明。<br>  - 选择发布策略(全量发布/灰度发布/定向推送)。<br>• **灰度发布**:新版本先向10%用户推送,观察数据(成功率、反馈),无问题后逐步扩大至100%。<br>• **强制更新vs可选更新**:<br>  - **强制更新**:发现重大bug或安全漏洞,自动为所有用户更新。<br>  - **可选更新**:功能优化类更新,用户可选择是否升级。<br>• **版本回滚**:新版本出现严重问题时,支持一键回滚到上一稳定版本。 |
| **3. 智能体下架与退役** | • **下架触发条件**:<br>  - 成功率持续低于阈值(如<80%)且无改进计划。<br>  - 严重安全漏洞或违规使用数据。<br>  - 开发团队主动申请下架。<br>• **下架流程**:<br>  - 提前通知所有订阅用户(如"该智能体将于7天后下架,请及时处理")。<br>  - 推荐替代智能体。<br>  - 下架后,历史使用记录保留可查询,但不可再调用。<br>• **智能体归档**:对于过时但仍有历史价值的智能体,标记为"已归档",不在商店展示但可通过直接链接访问。 |
| **4. 生态数据分析与运营** | • **生态健康度看板**:面向平台运营者,展示整体生态数据:<br>  - 智能体总数、活跃智能体数、新增/下架趋势。<br>  - 总订阅量、总调用量、整体满意度。<br>  - 各分类智能体的覆盖度和质量分布。<br>• **需求缺口分析**:<br>  - 分析用户搜索无结果的关键词,识别未被满足的需求。<br>  - 分析用户提交的需求,生成"最期待的10个智能体"列表,引导开发。<br>• **智能体推荐策略优化**:<br>  - 基于用户行为数据(浏览、订阅、使用、评价),优化推荐算法。<br>  - A/B测试不同推荐策略的效果。 |

### **三、核心业务流程示例**

#### **流程1:员工发现并订阅智能体**
1. **触发需求**:销售经理小王需要自动生成客户跟进邮件,节省时间。
2. **进入商店**:小王点击主工作台的"智能体市场"入口,进入商店首页。
3. **搜索与发现**:小王在搜索框输入"客户跟进",结果显示3个相关智能体,其中"智能邮件助手"评分4.8星、订阅数最多。
4. **查看详情**:小王点击进入详情页,查看功能说明:"基于客户画像和历史沟通,自动生成个性化跟进邮件",并观看演示视频。
5. **查看评价**:小王浏览用户评价,发现同事李总评论"使用后跟进效率提升50%,强烈推荐",增强信心。
6. **订阅**:小王点击"立即订阅",系统提示需授权"访问客户数据"和"发送邮件",小王确认授权,订阅成功。
7. **配置**:系统引导小王进行简单配置(设置邮件风格为"专业友好"、优先跟进"高价值客户"),完成。
8. **首次使用**:小王在主工作台对话框输入"为客户A生成跟进邮件",智能体自动查询客户A的画像和历史记录,3秒后生成个性化邮件草稿,小王略作修改后发送,非常满意,点赞👍。

#### **流程2:智能体开发者迭代优化**
1. **收到反馈**:"智能邮件助手"开发者在健康度看板中发现,最近7天用户反馈中有20%提到"邮件语气太生硬"。
