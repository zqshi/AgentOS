### **「智核」平台 - "智能知识库"模块详细产品需求文档**

**文档编号**:ZH-PRD-M2
**模块名称**:智能知识库
**产品定位**:超越传统Wiki的新一代智能化知识创作、管理与消费平台。
**核心目标**:通过AI深度赋能和知识图谱支撑,将碎片化、非结构化的信息转化为结构化、可关联、易复用的企业知识资产,大幅降低知识生产与消费的门槛。

---

### **一、 模块全景与核心价值**

#### **1.1 要解决的根本问题**
传统企业知识库面临四大核心挑战:
1. **创作成本高**:文档编写耗时,结构化整理困难,领域专家缺乏时间和动力沉淀知识。
2. **知识孤岛严重**:文档间缺乏有效关联,用户只能通过搜索找到单篇文档,无法发现相关知识网络。
3. **质量参差不齐**:缺乏统一术语规范,同一概念多种表述,新老文档冲突,用户难以判断可信度。
4. **消费体验差**:纯文本呈现,缺乏上下文,用户需要自行理解和提取关键信息。

#### **1.2 核心设计理念**
1. **AI助力生产**:让AI成为知识创作的得力助手,自动生成结构、推荐内容、规范术语。
2. **图谱驱动关联**:所有知识自动融入"灵枢"知识图谱,实现智能关联与网络化呈现。
3. **多视图适配**:既保留传统文档树满足用户习惯,又提供图谱视图、主题视图等创新体验。
4. **生命周期管理**:从草稿创建、协作评审、正式发布到归档废止,全流程状态可控。

#### **1.3 核心用户角色**
| 角色 | 职责与目标 | 核心使用场景 |
| :--- | :--- | :--- |
| **知识创作者** | 高效产出高质量、结构化的知识文档。 | 撰写技术文档、编写最佳实践、整理项目复盘。 |
| **领域专家** | 将隐性经验显性化,建立领域知识体系。 | 构建专业知识图谱、审核专业内容、指导新人。 |
| **知识消费者** | 快速找到并理解所需知识,应用到实际工作中。 | 查阅文档、学习流程、参考案例、发现关联知识。 |
| **知识管理员** | 维护知识库的质量、一致性与组织结构。 | 审核发布、整理分类、处理冲突、管理权限。 |

### **二、 详细功能需求**

#### **FR2.1 多视图知识浏览体系**

**目标**:提供多样化的知识组织与呈现方式,适应不同用户习惯与使用场景。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 文档树视图(经典模式)** | • **文件夹层级结构**:提供传统的树状目录结构,支持多级文件夹嵌套。<br>• **拖拽组织**:支持通过拖拽移动文档或文件夹,调整组织结构。<br>• **快速导航**:<br>  &nbsp;&nbsp;- 支持文件夹收藏,常用目录固定在侧边栏顶部。<br>  &nbsp;&nbsp;- 支持面包屑导航,快速跳转上级目录。<br>  &nbsp;&nbsp;- 支持树内搜索,快速定位文档。<br>• **批量操作**:支持批量移动、批量设置权限、批量导出等。 |
| **2. 知识图谱视图(创新模式)** | • **交互式图谱画布**:以可视化图形展示知识网络:<br>  &nbsp;&nbsp;- **节点**:代表知识实体(文档、概念、项目、人员等)。<br>  &nbsp;&nbsp;- **连线**:代表实体间的关系(引用、依赖、相关、作者等)。<br>• **智能布局**:<br>  &nbsp;&nbsp;- 自动计算节点位置,核心知识居中,相关知识围绕分布。<br>  &nbsp;&nbsp;- 支持力导向布局、层次布局、环形布局等多种算法切换。<br>• **交互探索**:<br>  &nbsp;&nbsp;- 点击节点展开其直接关联的知识。<br>  &nbsp;&nbsp;- 支持多跳展开(如"显示该节点的2度关系网络")。<br>  &nbsp;&nbsp;- 鼠标悬停节点显示快速预览(标题、摘要、关键指标)。<br>  &nbsp;&nbsp;- 支持按关系类型筛选(如"仅显示引用关系")。<br>• **路径分析**:选择两个节点,自动高亮它们之间的最短关联路径。 |
| **3. 主题/项目视图(业务模式)** | • **主题空间**:按业务域、产品线、项目等维度创建主题空间。<br>• **自动聚合**:系统根据文档标签、关联关系、作者团队,自动归集相关知识到对应主题。<br>• **空间概览**:<br>  &nbsp;&nbsp;- **知识地图**:可视化展示该主题下的知识结构与覆盖度。<br>  &nbsp;&nbsp;- **核心文档**:智能识别该主题的基础文档、高频访问文档。<br>  &nbsp;&nbsp;- **关键人物**:该主题的核心贡献者与专家。<br>  &nbsp;&nbsp;- **活跃动态**:最近更新、讨论热点。 |
| **4. 时间轴视图(演进模式)** | • **知识演进追踪**:以时间轴形式展示某个主题或项目下知识的产生与演进历史。<br>• **里程碑标注**:重要版本、重大决策、关键事件在时间轴上突出显示。<br>• **对比分析**:选择时间轴上的两个时间点,对比知识体系的变化。 |

#### **FR2.2 AI辅助创作空间**

**目标**:让知识创作从痛苦的劳动变为流畅的智能协作,大幅降低结构化知识生产成本。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 富文本多模态编辑器** | • **所见即所得编辑**:提供现代化的富文本编辑器(类Notion体验):<br>  &nbsp;&nbsp;- 支持标题层级、列表、引用、代码块、表格。<br>  &nbsp;&nbsp;- 支持Markdown快捷语法(如`#`生成标题、`-`生成列表)。<br>• **多媒体嵌入**:<br>  &nbsp;&nbsp;- 支持插入图片、视频、音频,自动转码与压缩。<br>  &nbsp;&nbsp;- 支持嵌入外部链接预览卡片(如会议链接、外部文档)。<br>  &nbsp;&nbsp;- 支持插入图表(柱状图、折线图、饼图),数据源可关联"灵枢"数据。<br>• **思维导图与流程图**:<br>  &nbsp;&nbsp;- 内置思维导图编辑器,可一键转换为文档大纲。<br>  &nbsp;&nbsp;- 内置流程图/泳道图编辑器,用于流程说明。<br>• **协作编辑**:支持多人实时协作编辑,显示在线用户光标与选中区域。 |
| **2. AI写作助手** | • **智能大纲生成**:<br>  &nbsp;&nbsp;- 用户输入文档标题和类型(如"技术方案"、"项目复盘"、"产品需求"),AI自动生成结构化大纲。<br>  &nbsp;&nbsp;- 大纲可编辑调整,确认后作为文档骨架。<br>• **段落续写与改写**:<br>  &nbsp;&nbsp;- 选中段落,点击"续写",AI基于上下文生成后续内容。<br>  &nbsp;&nbsp;- 选中段落,点击"改写",AI提供改写建议(如"更专业"、"更简洁"、"更通俗")。<br>• **内容建议**:<br>  &nbsp;&nbsp;- 编辑过程中,侧边栏实时展示相关建议:<br>    &nbsp;&nbsp;&nbsp;&nbsp;• **相关案例**:从"灵枢"图谱中检索的类似场景、历史案例。<br>    &nbsp;&nbsp;&nbsp;&nbsp;• **引用推荐**:可能需要引用的相关文档、数据。<br>    &nbsp;&nbsp;&nbsp;&nbsp;• **专家推荐**:该主题的领域专家,建议@其审阅。<br>• **自动摘要生成**:<br>  &nbsp;&nbsp;- 文档完成后,一键生成多粒度摘要(一句话摘要、三要点摘要、200字摘要)。 |
| **3. 智能关联与标注** | • **自动实体识别**:<br>  &nbsp;&nbsp;- 编辑器自动识别文本中的本体实体(如项目名、产品名、技术术语),以不同颜色下划线标注。<br>  &nbsp;&nbsp;- 点击实体可查看其在"灵枢"本体中的定义,或跳转到相关知识。<br>• **智能链接插入**:<br>  &nbsp;&nbsp;- 输入`[[`触发智能链接,搜索并插入内部知识文档链接。<br>  &nbsp;&nbsp;- 链接显示为卡片形式,包含目标文档标题、摘要、更新时间。<br>• **术语规范检查**:<br>  &nbsp;&nbsp;- 实时扫描文档,当使用非标准术语或同义词时,给出提示并建议替换为本体中的标准术语。<br>  &nbsp;&nbsp;- 标红未定义的专有名词,建议作者补充定义或关联已有概念。<br>• **知识图谱直接编辑**:<br>  &nbsp;&nbsp;- 侧边栏显示当前文档在知识图谱中的位置与关联。<br>  &nbsp;&nbsp;- 支持直接拖拽添加新的关联关系(如"此文档依赖XX项目")。<br>  &nbsp;&nbsp;- 新增关系自动同步到"灵枢"知识图谱。 |
| **4. 模板与组件库** | • **文档模板库**:提供丰富的预制模板:<br>  &nbsp;&nbsp;- **通用模板**:会议纪要、项目复盘、技术方案、产品需求。<br>  &nbsp;&nbsp;- **行业模板**:根据企业行业特点定制的专业模板。<br>• **可复用组件**:支持将常用内容块(如免责声明、审批流程说明)保存为组件,插入时自动填充。<br>• **个人/团队模板**:用户可创建私有或团队共享的自定义模板。 |

#### **FR2.3 版本管理与协作评审**

**目标**:确保知识质量,支持团队协作,保留完整演进历史。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 自动版本控制** | • **版本快照**:<br>  &nbsp;&nbsp;- 每次保存自动创建版本快照(类似Git commit)。<br>  &nbsp;&nbsp;- 支持手动创建重要版本节点(如V1.0正式版)并添加版本说明。<br>• **版本历史查看**:<br>  &nbsp;&nbsp;- 侧边栏展示文档的完整版本历史(时间、作者、变更说明)。<br>  &nbsp;&nbsp;- 支持点击任意历史版本进行预览。<br>• **版本对比**:<br>  &nbsp;&nbsp;- 选择任意两个版本进行可视化对比,高亮显示新增、删除、修改内容。<br>  &nbsp;&nbsp;- 支持段落级和字符级的差异展示。<br>• **版本回滚**:支持一键回滚到任意历史版本,回滚操作本身也会被记录为新版本。 |
| **2. 评审与评论系统** | • **段落级评论**:<br>  &nbsp;&nbsp;- 选中文档中的任意文本,点击添加评论。<br>  &nbsp;&nbsp;- 评论以标注形式附着在原文旁,不影响正文阅读。<br>  &nbsp;&nbsp;- 支持评论的回复、@提及、标记已解决。<br>• **评审工作流**:<br>  &nbsp;&nbsp;- 作者可发起评审,指定评审人(@领域专家或团队成员)。<br>  &nbsp;&nbsp;- 评审人收到通知,进入评审模式,可添加评论、建议修改。<br>  &nbsp;&nbsp;- 作者根据反馈修改,标记评论为"已处理"。<br>  &nbsp;&nbsp;- 所有评审人通过后,文档可进入"待发布"状态。<br>• **讨论串关联**:<br>  &nbsp;&nbsp;- 评论讨论作为知识的附加维度,与文档关联存储。<br>  &nbsp;&nbsp;- 讨论中的重要结论可一键提取为文档内容或FAQ。 |
| **3. 状态与生效管理** | • **文档状态流转**:文档具有明确的状态机制:<br>  &nbsp;&nbsp;- **草稿**:编辑中,仅作者和协作者可见。<br>  &nbsp;&nbsp;- **评审中**:已提交评审,评审人可见。<br>  &nbsp;&nbsp;- **已发布**:正式生效,按权限向全员可见。<br>  &nbsp;&nbsp;- **已归档**:不再维护,但保留可查询。<br>  &nbsp;&nbsp;- **已废止**:明确标记为无效,不再推荐使用。<br>• **生效范围设置**:<br>  &nbsp;&nbsp;- 作者可设置文档的生效范围(全公司/特定部门/特定团队/特定项目)。<br>  &nbsp;&nbsp;- 设置有效期(如"仅在2025年Q1有效"),到期自动标记提醒。<br>• **权限控制**:<br>  &nbsp;&nbsp;- 支持细粒度权限设置(查看/评论/编辑/管理)。<br>  &nbsp;&nbsp;- 支持继承父文件夹权限或独立设置。 |

#### **FR2.4 知识质量保障**

**目标**:通过自动化工具和机制,持续提升知识库的准确性、一致性与可信度。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 质量自动检测** | • **完整性检查**:<br>  &nbsp;&nbsp;- 检测文档是否包含必要章节(如技术方案是否包含"背景"、"方案"、"风险")。<br>  &nbsp;&nbsp;- 检测关键属性是否填写(如作者、标签、所属项目)。<br>• **一致性检查**:<br>  &nbsp;&nbsp;- 检测文档内术语使用是否一致。<br>  &nbsp;&nbsp;- 检测文档与本体定义是否冲突。<br>• **时效性检查**:<br>  &nbsp;&nbsp;- 检测文档中引用的外部链接是否失效。<br>  &nbsp;&nbsp;- 检测文档是否长期未更新(如超过1年),提示复审。<br>• **可读性评估**:<br>  &nbsp;&nbsp;- 分析文档的可读性得分(基于句子长度、专业术语密度等)。<br>  &nbsp;&nbsp;- 对过于复杂的段落给出简化建议。 |
| **2. 智能去重与冲突提示** | • **相似文档识别**:<br>  &nbsp;&nbsp;- 创建新文档时,系统自动检索是否存在相似或重复内容。<br>  &nbsp;&nbsp;- 提示作者:"发现相似文档《XX》,是否查看或合并?"<br>• **内容冲突检测**:<br>  &nbsp;&nbsp;- 检测知识库中是否存在对同一事物的不一致描述。<br>  &nbsp;&nbsp;- 向知识管理员报告冲突,引导发起仲裁或统一修订。 |
| **3. 知识评分与信誉体系** | • **文档质量评分**:基于多维度计算文档质量分:<br>  &nbsp;&nbsp;- **内容质量**:完整性、结构化程度、术语规范性。<br>  &nbsp;&nbsp;- **受欢迎度**:浏览量、点赞数、收藏数。<br>  &nbsp;&nbsp;- **权威性**:作者专业度、引用次数。<br>  &nbsp;&nbsp;- **时效性**:创建与更新时间。<br>• **展示与激励**:<br>  &nbsp;&nbsp;- 搜索和推荐时,质量评分影响排序权重。<br>  &nbsp;&nbsp;- 高分文档获得"优质内容"标识,作者获得积分奖励。 |

### **三、 核心业务流程示例**

**流程:领域专家创建并发布一篇技术方案文档**

1. **创建文档**:
   - 用户点击"新建文档",选择"技术方案"模板。
   - AI自动生成方案大纲(背景、目标、方案设计、风险评估等)。

2. **AI辅助写作**:
   - 用户填写"背景"章节时,侧边栏推荐相关历史项目案例。
   - 用户一键引用某个案例,自动插入智能链接。
   - 编辑器自动识别出"微服务架构"、"Kubernetes"等术语,并与本体关联。

3. **实时质量检查**:
   - 系统检测到用户使用了非标准术语"K8S",提示替换为标准术语"Kubernetes"。
   - 用户完成初稿后,系统提示"方案设计"章节内容较少,建议补充。

4. **协作评审**:
   - 用户点击"发起评审",@架构师和项目经理。
   - 评审人收到通知,添加评论:"方案中的缓存策略需要更详细的说明"。
   - 用户补充内容后,标记评论为"已解决"。

5. **发布与关联**:
   - 评审通过后,用户设置文档状态为"已发布",生效范围为"技术部门"。
   - 系统自动将文档关联到"XX项目"主题空间,并在知识图谱中建立与相关技术、项目的连接。

6. **持续维护**:
   - 半年后,系统检测到文档引用的某个API已废弃,向作者发送提示。
   - 作者更新文档,系统自动创建新版本并通知关注该文档的用户。

### **五、 非功能性需求**

| 需求类别 | 具体要求 |
| :--- | :--- |
| **性能** | • 文档打开速度 < 1秒(10万字以内)。<br>• 编辑器输入响应延迟 < 100ms,无卡顿感。<br>• 知识图谱视图(100节点)加载与交互流畅,帧率 ≥ 30fps。<br>• AI写作助手响应 < 3秒。 |
| **可靠性** | • 编辑器支持自动保存(每30秒或每次关键操作后)。<br>• 网络中断后,用户编辑内容暂存本地,恢复后自动同步。<br>• 版本数据异地备份,确保历史版本不丢失。 |
| **体验** | • 编辑器界面简洁,功能按钮布局合理,学习成本低。<br>• 加载过程使用骨架屏,避免白屏等待。<br>• 操作提供即时反馈(如保存成功提示、评审状态更新)。 |
| **兼容性** | • 支持导入常见格式(Word、Markdown、PDF文本提取)。<br>• 支持导出为Word、PDF、Markdown、HTML。<br>• 适配主流浏览器和屏幕尺寸。 |
