## **「智核」平台 - "团队空间与项目协作"模块详细产品需求文档**

**文档编号**:ZH-PRD-M4
**模块名称**:团队空间与项目协作
**产品定位**:为团队和项目提供智能化的协作工作区,自动沉淀团队知识资产,促进协作效率与知识传承。
**核心目标**:打破部门和项目的知识孤岛,让团队协作过程中产生的所有智慧(讨论、决策、文档、经验)自动结构化沉淀,形成可复用、可传承的团队智慧资产,并通过智能化工具提升协作效率。

---

## **产品边界澄清：团队空间不是什么**

### **团队空间 ≠ 项目管理工具**
- **不替代** Jira/Asana/Monday/禅道等项目管理系统
- **不提供** 完整的项目规划、甘特图、资源调度、燃尽图、Sprint管理
- **不做** 复杂的任务依赖管理、工时统计、风险管理

### **团队空间 ≠ IM群组**
- **不替代** 企业微信群/钉钉群/Slack频道/Teams频道
- **不提供** 实时聊天、群视频、语音通话、文件传输
- **不做** 即时消息推送、@所有人、群公告

### **团队空间 = 知识协作层**
- **是** 团队知识资产的聚合中心（文档、决策、讨论、会议纪要）
- **是** 从IM/会议/项目管理系统中**自动提取**协作知识并结构化
- **是** 让团队智慧沉淀、可追溯、可复用的空间
- **通过集成** 从外部系统获取协作数据 → AI智能加工 → 结构化沉淀 → 主动推送

### **与外部系统的集成方式**

| 外部系统 | 职责边界 | 团队空间的集成方式 |
|:---|:---|:---|
| **IM工具**（企业微信/钉钉/飞书） | 实时通讯、群聊、即时消息 | **单向数据获取**：<br>• 通过API/Webhook获取群聊中的讨论记录<br>• AI识别有价值的讨论片段（决策、问答、方案）<br>• 自动提取为结构化知识并关联到团队空间<br>• 智核的通知可推送到IM群 |
| **会议系统**（Zoom/Teams/腾讯会议） | 音视频会议、屏幕共享、会议室管理 | **会议内容获取**：<br>• 获取会议录音/录屏/聊天记录/白板<br>• AI自动生成会议纪要、提取决策和待办<br>• 归档到团队空间的"会议室"<br>**不做**：不提供会议调度和视频能力 |
| **项目管理工具**（Jira/禅道） | 需求管理、任务分配、Sprint规划、进度跟踪 | **任务状态同步**：<br>• 从Jira同步任务状态到团队空间看板<br>• 完成状态可回写到Jira<br>• 任务相关讨论和知识自动关联<br>**不做**：不提供完整项目管理能力 |
| **文档协作**（飞书文档/石墨） | 实时协同编辑、权限管理 | **文档导入与关联**：<br>• 支持从协作平台导入文档<br>• 文档讨论和批注同步<br>**不做**：不完全替代协作编辑工具 |

**核心原则**：团队空间是**知识沉淀层**，通过集成从各系统**捕获协作痕迹** → **AI智能加工** → **结构化存储** → **主动推送与发现**

### **一、模块全景与核心价值**

#### **1.1 要解决的根本问题**
团队协作中存在三大知识与效率痛点:
1. **协作知识散落**:团队协作产生的知识分散在邮件、IM群聊、会议、各种文档中,缺乏统一沉淀,项目结束后知识流失严重。
2. **缺乏全景视图**:团队成员难以快速了解项目全貌(目标、进展、决策历史、关键文档、团队成员),信息同步成本高。
3. **隐性专家难发现**:团队中谁擅长什么、谁对某个话题最有发言权,往往依赖人际网络和主管了解,新成员融入困难。

团队空间模块通过**专属空间聚合**、**自动知识沉淀**、**专家发现**三大能力,将团队协作从"临时拼凑"升级为"智能有机体"。

#### **1.2 核心设计理念**
1. **空间即上下文 (Space as Context)**:每个团队/项目拥有独立空间,空间内的所有活动自动关联,形成完整上下文。
2. **协作即沉淀 (Collaboration as Knowledge)**:会议、讨论、文档编辑等协作行为自动转化为结构化知识,而非事后整理。
3. **透明化智慧 (Transparent Intelligence)**:团队的知识网络、成员贡献、隐性专家通过可视化方式透明呈现。
4. **进化型空间 (Evolvable Space)**:空间不是静态容器,而是随团队工作动态演化,自动推荐优化建议。

#### **1.3 核心用户角色**
| 角色 | 核心使用场景 | 关键价值获得 |
| :--- | :--- | :--- |
| **项目负责人/团队Leader** | 创建和管理团队空间;监控项目进展;促进知识沉淀;管理成员权限。 | 获得项目全景仪表盘;自动化知识管理;快速发现团队能力地图。 |
| **项目成员** | 在空间中协作编辑文档;参与讨论;查找项目相关资料;了解决策背景。 | 一站式获取所需信息;避免重复沟通;快速融入项目上下文。 |
| **新加入成员** | 快速了解项目背景、目标、历史决策、关键文档、团队成员。 | 通过智能导览快速上手;自动推荐学习路径;识别可求助的专家。 |
| **跨团队协作者** | 需要了解其他团队的工作输出和经验;寻找可复用的方案和案例。 | 跨空间知识发现;快速找到对口专家;避免重复造轮子。 |
| **组织管理者** | 了解各团队健康度;识别高绩效团队的最佳实践;促进跨团队知识流动。 | 获得团队知识资产评估报告;发现可复制的协作模式;优化资源配置。 |

### **二、详细功能需求**

#### **M4.1 专属团队空间门户**

**目标**:为每个团队/项目创建独立、智能的协作工作区,聚合所有相关资源,提供全景视图。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 空间创建与配置** | • **空间类型**:支持创建不同类型的空间:<br>  - **项目空间**:有明确开始和结束时间的临时协作(如"Q1新产品开发")。<br>  - **团队空间**:长期存在的组织单元(如"研发中心"、"销售华东区")。<br>  - **主题空间**:跨团队的主题协作(如"数字化转型委员会"、"技术创新小组")。<br>• **空间元信息**:<br>  - **基础信息**:空间名称、描述、封面、标签、所属部门。<br>  - **时间信息**:创建时间、预计结束时间(项目空间)、里程碑节点。<br>  - **目标与OKR**:可设置空间的核心目标和关键结果(支持结构化表单)。<br>• **空间模板**:提供预置模板(敏捷开发项目、市场活动、研究课题),一键创建并自动配置常用结构和工具。 |
| **2. 空间首页与仪表盘** | • **智能概览卡片**:首页以卡片形式展示关键信息:<br>  - **目标与进度**:OKR完成度、里程碑状态(用进度条或燃尽图可视化)。<br>  - **最新动态**:近期文档更新、讨论、决策、成员变更,按时间流展示。<br>  - **待办事项**:空间内分配给"我"的任务,按优先级排序。<br>  - **关键文档**:自动识别的核心文档(如项目计划、需求文档),置顶展示。<br>  - **团队成员**:成员列表(头像墙),标注角色和在线状态。<br>• **快捷入口**:<br>  - 提供"新建文档"、"发起讨论"、"查看知识图谱"、"查看会议记录"等常用操作快捷按钮。<br>• **个性化视图**:每个成员看到的首页可根据其角色和权限自动调整(如Leader看到全局统计,成员看到与自己相关的任务和文档)。 |
| **3. 空间导航与结构** | • **多维度组织**:空间内容按多个维度组织:<br>  - **文档库**:文件夹树形结构,存放所有项目文档。<br>  - **讨论区**:话题式讨论板,支持分类(技术讨论/需求澄清/风险问题)。<br>  - **会议室**:所有会议记录的归档区。<br>  - **知识图谱**:该空间的专属知识子图。<br>  - **成员目录**:成员列表及其贡献统计。<br>• **智能搜索**:空间内全局搜索,范围限定在本空间,结果按相关度和时效性排序。 |

#### **M4.2 自动知识沉淀引擎**

**目标**:让团队协作过程中产生的碎片化智慧,自动转化为结构化、可复用的知识资产,零额外成本。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 会议智能记录与提炼** | • **会议关联**:创建会议时,选择关联的团队空间,会议所有产出自动归入该空间。<br>• **实时转录与摘要**:<br>  - 在线会议自动录音转文字(支持接入腾讯会议、钉钉、Zoom等)。<br>  - AI实时生成会议摘要(参会人、议题、关键讨论点、决策、待办事项)。<br>• **结构化提取**:会议结束后,AI自动提取:<br>  - **决策事项**:明确的决策结论(如"确定采用方案B"),自动标记并关联到相关文档或任务。<br>  - **待办任务**:会议中分配的任务(如"@张三 下周完成原型设计"),自动生成任务卡片并推送给责任人。<br>  - **风险与问题**:讨论中提及的风险或未解决问题,自动提取并加入"问题跟踪列表"。<br>  - **知识点**:有价值的经验分享或技术方案讨论,提取为知识卡片,建议发布到知识库。<br>• **会议关联网络**:自动识别会议间的关联(如"本次会议是上次的后续"、"与XX项目相关"),在知识图谱中建立连接。 |
| **2. 讨论区智能沉淀** | • **话题式讨论**:<br>  - 成员可在空间讨论区发起话题讨论(类似论坛帖子),支持富文本、附件、@提及。<br>  - 话题可分类(技术、需求、问题、分享)并打标签,便于组织和检索。<br>• **讨论智能提炼**:<br>  - 对于长讨论串,AI自动生成"讨论摘要"(核心观点、分歧点、最终结论)。<br>  - 识别讨论中的"最佳答案"(基于点赞、采纳、参与者专业度),标注并置顶。<br>• **从讨论到知识**:<br>  - 讨论中的重要结论可一键"转为文档"或"添加到FAQ",自动生成初稿。<br>  - 讨论自动关联到相关文档和知识图谱实体。<br>• **讨论订阅与通知**:成员可订阅关注的话题分类,有新讨论或回复时自动通知。 |
| **3. 文档自动聚合与关联** | • **文档归属**:在空间内创建的所有文档,自动归属于该空间,并打上空间标签。<br>• **智能分类**:AI根据文档内容,自动推荐分类和存放位置(如"这是需求文档,建议放入'需求管理'文件夹")。<br>• **跨空间引用追踪**:当空间内文档被其他空间或个人引用时,自动记录并在文档属性中展示"被引用关系"。<br>• **文档关联推荐**:编辑文档时,右侧栏自动推荐空间内其他相关文档(基于语义相似度和共现关系)。 |
| **4. 决策与里程碑记录** | • **决策日志**:<br>  - 空间提供"决策日志"功能,重大决策需正式记录(决策内容、背景、参与者、时间、依据)。<br>  - 决策可从会议记录、讨论中自动提取,或手动创建。<br>  - 决策在知识图谱中作为关键节点,关联相关文档、会议、任务。<br>• **里程碑管理**:<br>  - 项目空间可设置里程碑节点(如"需求评审完成"、"Alpha版本发布")。<br>  - 里程碑达成时,系统自动创建快照(记录此时空间状态、关键文档版本),用于回溯和复盘。 |

#### **M4.3 团队智慧可视化**

**目标**:让团队的知识网络、成员贡献、隐性专家透明可见,促进知识流动与能力识别。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 空间知识图谱** | • **专属子图**:每个空间拥有独立的知识子图,展示该空间内所有知识实体及其关联:<br>  - **节点类型**:文档、会议、讨论、决策、成员、外部引用。<br>  - **关系类型**:引用、依赖、参与、衍生、相似。<br>• **可视化探索**:<br>  - 力导向图展示,核心概念居中,相关知识围绕分布。<br>  - 点击节点查看详情和关联内容,支持多跳展开。<br>  - 按时间轴过滤,查看知识网络的演进过程(如"项目第1个月vs第6个月的知识图谱对比")。<br>• **知识密度分析**:识别知识密集区域(高度关联的知识簇)和知识空洞(缺乏覆盖的领域),给出补充建议。 |
| **2. 团队专家发现与画像** | • **自动专家识别**:基于成员在空间内的行为数据,自动识别各领域的"隐性专家":<br>  - **贡献维度**:创建文档数量、质量评分、被引用次数。<br>  - **互动维度**:讨论参与度、回答采纳率、会议发言质量。<br>  - **专业维度**:涉及的专业领域(基于本体标签分析)。<br>• **成员专业标签云**:为每个成员生成专业标签云(如"张三:微服务架构、Kubernetes、性能优化"),标签大小代表专业度。<br>• **专家推荐**:当成员提问或寻求帮助时,系统自动推荐最合适的团队专家。<br>• **成员贡献看板**:<br>  - 展示每个成员的贡献统计(文档数、讨论参与、知识被引用数)。<br>  - 可视化"贡献雷达图"(创作、评审、讨论、协作等多维度)。<br>  - 设置"月度MVP"自动评选,基于综合贡献度。 |
| **3. 知识流向与影响力分析** | • **知识传播路径**:追踪知识在空间内的传播路径(如"文档A → 被会议B引用 → 启发决策C → 衍生文档D")。<br>• **知识影响力评分**:为空间内每个知识对象计算影响力分数(基于引用次数、讨论热度、应用广度)。<br>• **知识贡献排行**:展示最具影响力的文档、最活跃的讨论、最关键的决策,帮助识别团队智慧的核心资产。 |

#### **M4.4 团队协作增强工具**

**目标**:提供智能化工具,降低协作摩擦,提升团队效率。

| 功能组件 | 详细需求描述 |
| :--- | :--- |
| **1. 智能任务协同** | • **任务看板**:空间内提供共享任务看板(看板视图或列表视图),展示所有团队任务。<br>• **任务来源整合**:<br>  - 会议中分配的任务自动生成。<br>  - 文档评审产生的待办自动生成。<br>  - 成员手动创建的任务。<br>• **任务属性**:标题、描述、责任人、优先级、截止日期、关联文档/会议、状态(待办/进行中/已完成/阻塞)。<br>• **智能提醒**:<br>  - 临近截止时间自动提醒责任人。<br>  - 任务阻塞超过设定时间,自动@团队Leader。<br>• **任务依赖管理**:支持设置任务间的依赖关系(如"任务B依赖任务A完成"),可视化为甘特图。 |
| **2. 新成员智能导览** | • **欢迎页与入职指引**:新成员加入空间时,自动展示欢迎页:<br>  - **空间介绍**:目标、当前进展、关键里程碑。<br>  - **团队成员**:成员列表、角色、联系方式。<br>  - **必读文档**:AI自动推荐的核心文档清单(如项目计划、技术架构)。<br>• **学习路径生成**:AI基于空间知识图谱,为新成员生成推荐学习路径(先读什么、再读什么)。<br>• **专家对接**:自动推荐可以求助的团队专家(基于新成员的背景和需要了解的领域)。<br>• **融入度追踪**:系统追踪新成员的融入进度(已阅读文档数、参与讨论数、提问数),给Leader提供反馈。 |
| **3. 跨空间协作与知识复用** | • **空间互联**:支持在空间间建立关联(如"空间A是空间B的子项目"、"空间C与空间D有协作关系")。<br>• **跨空间搜索**:支持在有权限的多个空间内同时搜索,发现可复用的知识和方案。<br>• **最佳实践推荐**:当创建新空间时,系统自动推荐相似历史空间的最佳实践(如"类似项目的成功经验"、"常见坑点")。<br>• **知识订阅**:成员可订阅其他空间的特定标签或主题,有更新时自动推送。 |
| **4. 空间健康度监控** | • **健康度评分**:系统自动为空间计算健康度评分,综合考虑:<br>  - **活跃度**:成员参与频率、文档更新频率。<br>  - **知识完整性**:核心文档是否齐全、知识覆盖广度。<br>  - **协作质量**:讨论深度、决策记录完整性。<br>  - **进度匹配度**:里程碑达成情况vs计划。<br>• **异常预警**:<br>  - 空间长期无活动(如2周无更新)→ 预警"空间可能停滞"。<br>  - 知识覆盖度低(如只有10篇文档)→ 建议"补充关键文档"。<br>  - 成员贡献极度不均(如80%工作由1人完成)→ 提示"团队协作失衡"。<br>• **优化建议**:基于健康度分析,AI给出具体改进建议(如"建议每周召开同步会议"、"建议完善需求文档")。 |

### **三、核心业务流程示例**

#### **流程1:新项目从创建到知识沉淀全周期**
1. **项目启动**:产品经理创建"Q1新产品开发"项目空间,选择"敏捷开发项目"模板,填写项目目标和里程碑。
2. **团队组建**:添加团队成员(研发、设计、测试、运营),系统为每个成员推送欢迎消息和必读文档清单。
3. **需求讨论**:团队在讨论区发起"核心功能优先级"话题,10条讨论后,AI自动生成讨论摘要,产品经理点击"转为文档",生成"需求优先级决策文档"。
4. **会议决策**:召开"技术方案评审会",会议中决定采用微服务架构,系统自动提取决策,创建"决策001:采用微服务架构"日志,关联会议记录和技术方案文档。
5. **任务分配**:会议中分配任务"@张三 完成API设计",系统自动生成任务卡片,推送给张三,并在空间任务看板中显示。
6. **文档协作**:张三创建"API设计文档",AI推荐相关历史文档"微服务最佳实践",张三引用后,系统自动在知识图谱中建立关联。
7. **项目进行中**:空间首页实时更新进度,健康度监控发现"测试文档缺失",预警并@测试负责人。
8. **项目复盘**:项目结束,Leader点击"生成项目复盘报告",AI自动汇总项目数据(里程碑达成情况、关键决策回顾、成员贡献统计、知识产出清单),生成结构化报告。
9. **知识沉淀**:复盘报告发布到知识库,空间知识图谱自动归入企业知识网络,团队专家标签更新,未来相似项目可复用。

#### **流程2:新成员快速融入团队**
1. **加入空间**:新员工小李被添加到"销售运营团队"空间,收到欢迎消息和自动生成的"入职导览"。
2. **学习路径**:系统推荐学习路径:"先读'销售流程SOP' → 再看'客户案例库' → 最后了解'常见问题处理'"。
3. **专家对接**:系统推荐"@王经理是客户关系管理专家,可向其请教",小李主动联系。
4. **参与讨论**:小李在讨论区提问"大客户折扣审批流程",资深成员快速回答并@相关文档,讨论标记为"已解决"。
5. **融入追踪**:两周后,系统向Leader推送"小李融入度报告:已阅读80%核心文档,参与5次讨论,建议安排一对一辅导"。

#### **流程3:跨团队知识复用**
1. **需求场景**:研发团队A遇到"分布式缓存架构"难题,Leader在主工作台搜索。
2. **跨空间发现**:搜索结果显示研发团队B的空间内有相关讨论和技术方案文档。
3. **知识复用**:团队A申请访问权限,查看团队B的方案,发现可直接借鉴,点击"复制到我的空间"。
4. **专家连接**:系统识别团队B的张工是该领域专家,推荐团队A向其咨询,促成跨团队协作。
5. **知识关联**:系统自动在知识图谱中建立"团队A的方案衍生自团队B",记录知识复用路径。
