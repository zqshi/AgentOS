## **「智核」平台 - "知识协作与质量保障"模块详细产品需求文档**

**文档编号**: ZH-PRD-M6
**模块名称**: 知识协作与质量保障（原"协作与审阅工作流"）
**产品定位**: 协作式知识生产助手，而非审批式质量控制
**核心目标**: 让知识在协作中自然进化，通过智能辅助和同行反馈持续提升质量，而非通过审批卡点控制发布

---

## **设计理念革新：从"审批控制"到"协作进化"**

### **❌ 传统做法的问题**
- 多层审批流程：草稿 → 审阅 → 批准 → 发布
- 卡点控制：未经批准不能发布，阻塞知识流动
- 责任转嫁：审批人成为质量背锅者，创作者反而不负责
- **结果**：知识生产周期长、积压严重、员工不愿贡献

### **✅ 新理念：Wiki式协作 + AI质量助手**

**核心原则**：
1. **默认开放**：知识创建后立即可见（至少团队可见），无需等待审批
2. **持续演进**：通过版本追踪，允许知识逐步完善，而非一次性完美
3. **社区监督**：依靠同行评审和使用反馈，而非审批人把关
4. **AI辅助**：机器智能预检质量问题，减轻人工负担
5. **分层可见**：根据成熟度和风险级别，自动控制可见范围

---

## **一、模块全景与核心价值**

### **1.1 要解决的根本问题**

1. **创作门槛高**
   - 担心犯错不敢发布，导致知识锁在个人电脑
   - 追求完美主义，导致知识永远处于"草稿"状态

2. **反馈延迟长**
   - 传统审批流程周期长（数天到数周）
   - 反馈到达时，创作者已失去上下文和热情

3. **质量责任不清**
   - 审批人走形式，创作者认为"批准了就不是我的问题"
   - 知识过期无人更新，因为"不是我批准的我不改"

### **1.2 核心设计理念**

**1. 即发即见 (Publish Fast)**
- 知识创建完成即可发布到团队/项目空间
- 无需等待审批，允许不完美但有价值的内容先流动起来

**2. 协作进化 (Collaborative Evolution)**
- 通过批注、建议、协同编辑让知识持续改进
- 版本历史完整记录，可追溯，可回滚

**3. AI智能助手 (AI Quality Assistant)**
- 机器预检常见问题（术语、格式、事实错误）
- 减少人工审阅工作量，聚焦专业判断

**4. 分层可见 (Layered Visibility)**
- 根据知识成熟度和风险级别，自动控制可见范围：
  - **个人草稿**：仅自己可见
  - **团队协作**：团队成员可见可编辑
  - **部门发布**：部门内可见
  - **全员发布**：全公司可见（需满足质量标准）
  - **对外发布**：面向客户/公众（需合规审核）

**5. 社区监督 (Community Governance)**
- 任何人发现错误可提出修正
- 高频使用、高评分的知识自动提升权威度
- 长期无更新且质量下降的知识自动降级或归档

### **1.3 核心用户角色**

| 角色 | 核心使用场景 | 关键价值获得 |
|:---|:---|:---|
| **知识创作者** | 创建知识后立即发布；接收同事建议；持续迭代完善。 | 无需等待审批，快速发布；获得即时反馈；保留完整创作自主权。 |
| **知识贡献者** | 发现他人知识中的问题或可改进点；提出建议或直接编辑补充。 | 降低贡献门槛；每个人都能参与知识完善；贡献被记录和认可。 |
| **知识消费者** | 阅读知识时发现错误或不清晰；留下反馈帮助改进。 | 看到质量问题可及时反馈；推动知识持续改进；避免踩坑。 |
| **领域专家** | 关注特定领域知识质量；对专业内容进行把关和指导。 | 被动把关变主动指导；通过批注和建议传递专业知识；建立专业影响力。 |
| **合规/法务** | 对敏感内容（对外发布、涉及法律/财务）进行必要审核。 | 仅处理真正高风险内容；审核范围明确；有完整审计记录。 |

## **二、详细功能需求**

### **M6.1 智能协作批注系统**

**目标**：让反馈和建议精准、高效、促进改进而非阻塞

| 功能组件 | 详细需求描述 |
|:---|:---|
| **1. 内嵌式批注** | • **段落级批注**：<br>&nbsp;&nbsp;- 选中段落，右侧展开批注面板<br>&nbsp;&nbsp;- 批注类型：<br>&nbsp;&nbsp;&nbsp;&nbsp;• **疑问**："这里的数据来源是哪里？"<br>&nbsp;&nbsp;&nbsp;&nbsp;• **建议**："建议补充XX案例"<br>&nbsp;&nbsp;&nbsp;&nbsp;• **错误**："这个结论与实际不符"<br>&nbsp;&nbsp;&nbsp;&nbsp;• **赞赏**："这段写得很好！"<br>• **行内批注**：<br>&nbsp;&nbsp;- 选中具体词句添加针对性批注<br>&nbsp;&nbsp;- 批注标记以下划线+颜色区分严重程度<br>• **建议编辑**：<br>&nbsp;&nbsp;- 直接在文档中建议修改（类似Word修订模式）<br>&nbsp;&nbsp;- 显示删除（删除线）和新增（高亮）<br>&nbsp;&nbsp;- 创作者可一键接受/拒绝 |
| **2. 批注交互** | • **@提及**：批注中@相关人员，触发通知<br>• **批注讨论**：每条批注支持多轮对话，形成讨论线程<br>• **批注状态**：<br>&nbsp;&nbsp;- 待处理 / 处理中 / 已采纳 / 不采纳<br>&nbsp;&nbsp;- 创作者处理后标记状态并说明处理方式<br>• **批注统计**：显示总批注数、待处理数、参与人数 |
| **3. AI智能预检** | • **自动质量检查**（创作者保存时自动触发）：<br>&nbsp;&nbsp;- **术语规范性**："第15行使用了'客户'，企业标准术语是'购买方'"<br>&nbsp;&nbsp;- **事实准确性**："这个数据与知识图谱中的数据不一致，请确认"<br>&nbsp;&nbsp;- **格式一致性**："标题层级不规范，建议调整"<br>&nbsp;&nbsp;- **完整性检查**："引用了外部研究但未标注来源"<br>&nbsp;&nbsp;- **可读性评分**："段落过长（>500字），建议拆分"<br>• **AI建议采纳**：<br>&nbsp;&nbsp;- AI建议以特殊颜色标记，创作者可一键接受或忽略<br>&nbsp;&nbsp;- 采纳率数据反馈优化AI模型<br>• **AI批注聚类**：<br>&nbsp;&nbsp;- 自动将相似批注聚类（如"都是关于术语问题"）<br>&nbsp;&nbsp;- 帮助创作者批量处理同类问题 |
| **4. 协同编辑** | • **多人实时协作**（可选开启）：<br>&nbsp;&nbsp;- 多人同时编辑同一文档，实时同步<br>&nbsp;&nbsp;- 显示其他人的光标位置和正在编辑的段落<br>&nbsp;&nbsp;- 冲突自动合并或提示手动解决<br>• **协作模式**：<br>&nbsp;&nbsp;- **建议模式**：协作者只能建议修改，创作者决定是否采纳<br>&nbsp;&nbsp;- **直接编辑模式**：团队成员可直接编辑（Wiki模式） |

### **M6.2 知识成熟度与分层发布**

**目标**：根据知识的完善程度和风险级别，智能控制可见范围，取代传统审批

| 功能组件 | 详细需求描述 |
|:---|:---|
| **1. 知识成熟度分级** | • **自动成熟度评分**（AI实时计算）：<br>&nbsp;&nbsp;- **草稿 (Draft)**：刚创建，未经任何反馈<br>&nbsp;&nbsp;- **初稿 (Initial)**：创作者自评完成，但未经他人review<br>&nbsp;&nbsp;- **协作中 (In Review)**：有他人批注或建议，正在迭代<br>&nbsp;&nbsp;- **成熟 (Mature)**：经过多人review，批注大部分已解决，无严重问题<br>&nbsp;&nbsp;- **精品 (Featured)**：高使用量、高评分、专家认可<br>• **评分因子**：<br>&nbsp;&nbsp;- 批注数量和质量（已解决/未解决）<br>&nbsp;&nbsp;- 参与review人数和专业度<br>&nbsp;&nbsp;- 使用量和用户评分<br>&nbsp;&nbsp;- AI质量检查得分<br>&nbsp;&nbsp;- 更新频率（定期更新 vs 长期未更新） |
| **2. 分层可见性规则** | • **自动可见性策略**：<br>&nbsp;&nbsp;- **草稿**：仅创作者可见<br>&nbsp;&nbsp;- **初稿**：创作者可选择发布到：<br>&nbsp;&nbsp;&nbsp;&nbsp;• 团队空间（团队成员可见）<br>&nbsp;&nbsp;&nbsp;&nbsp;• 部门空间（部门可见）<br>&nbsp;&nbsp;&nbsp;&nbsp;• 全员空间（需达到"成熟"级别，或创作者明确承担责任）<br>&nbsp;&nbsp;- **成熟**：自动推荐到相关主题，全员搜索可见<br>&nbsp;&nbsp;- **精品**：首页推荐，外部分享<br>• **风险标记**：<br>&nbsp;&nbsp;- AI自动识别高风险内容（涉及财务数据、客户信息、对外承诺）<br>&nbsp;&nbsp;- 高风险内容即使成熟也不自动全员可见，需合规审核 |
| **3. 质量标准配置** | • **全员发布标准**（可由知识管理员配置）：<br>&nbsp;&nbsp;- 至少1位非创作者review过<br>&nbsp;&nbsp;- AI质量检查得分 > 80分<br>&nbsp;&nbsp;- 无未解决的"错误"级批注<br>&nbsp;&nbsp;- 创作者确认承担内容准确性责任<br>• **对外发布标准**（高门槛）：<br>&nbsp;&nbsp;- 必须经过合规/法务审核<br>&nbsp;&nbsp;- 至少2位专家背书<br>&nbsp;&nbsp;- 内容不包含敏感信息<br>• **标准豁免**：<br>&nbsp;&nbsp;- 紧急情况下，创作者可申请快速发布（需说明理由）<br>&nbsp;&nbsp;- 事后补充review和改进 |

**目标**：仅对真正高风险的内容进行人工审核，其他一切自动化

|:---|:---|
| **1. 风险识别** | • **AI风险扫描**：<br>&nbsp;&nbsp;- 识别敏感信息（客户数据、财务数据、商业机密）<br>&nbsp;&nbsp;- 识别合规风险（对外承诺、法律条款、安全规范）<br>&nbsp;&nbsp;- 识别品牌风险（负面表述、不当言论）<br>• **风险分级**：<br>&nbsp;&nbsp;- **高风险**：必须人工审核<br>&nbsp;&nbsp;- **中风险**：AI预警，创作者自查<br>&nbsp;&nbsp;- **低风险**：无需特殊处理 |

| **3. 审计与合规报告** | • **审计日志**：完整记录所有审核操作<br>• **合规报告**：自动生成合规审核报告（满足ISO/等保要求） |

## **三、核心流程对比**

### **旧模式：传统审批流程**
1. 创作者完成草稿
2. 提交审阅 → **卡点1**：等待审阅者响应（1-3天）
3. 审阅者批注 → 创作者修改
4. 再次提交 → **卡点2**：等待二次审阅（1-2天）
5. 提交批准 → **卡点3**：等待管理员批准（1-7天）
6. 最终发布（耗时：**5-12天**）

**问题**：效率低、积压严重、质量责任模糊

---

### **新模式：协作进化流程**
1. 创作者完成初稿
2. **AI预检**：10秒内返回质量建议
3. 创作者处理AI建议，发布到团队空间（**0等待**）
4. 团队成员看到新知识，异步批注和建议（**持续反馈**）
5. 创作者根据反馈持续迭代（**版本演进**）
6. 当知识达到"成熟"标准，自动推荐到全员（**无需审批**）
7. 社区监督：任何人发现问题随时反馈修正（**持续优化**）

**优势**：
- **发布速度**：从12天缩短到10分钟（**99%提速**）
- **质量保障**：从单点审批变为社区监督（**更可靠**）
- **责任明确**：创作者自始至终对内容负责（**不推诿**）

## **四、非功能需求**

### **4.1 性能要求**
- AI质量检查响应时间 < 10秒（5000字文档）
- 批注加载延迟 < 200ms
- 版本对比计算时间 < 1秒
- 支持单文档1000+条批注

### **4.2 易用性要求**
- 新用户首次使用批注功能，5分钟内掌握
- 创作者发布知识，全流程 < 5分钟（含AI检查）
- 移动端支持批注查看和简单回复

### **4.3 数据安全**
- 所有批注和版本历史不可删除（仅可归档）
- 审计日志保留7年，满足合规要求
- 敏感内容自动加密存储

## **六、成功指标**

### **效率指标**
- 知识发布周期：从平均7天缩短到1天内（**85%提升**）
